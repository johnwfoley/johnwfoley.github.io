Block vs Inline

He's Not Wide he's my brother.

A couple of things about Blocks

  • A Block takes all the width. It will fill the entire width of its parent
  • Since its as wide as can be it will naturally go below what ever came before it.
  • Like A lot if things its height is determined by what ever is in it.

  • but the chief take away is: its as wide as the screen so things can go above it or below it, but not next to it.

A couple of things about Inline Elements

  • They "flow" so they will fill in where there is space. If the previous element is NOT as wide as the page and inline item will start where it ends. It won't clear to the next line.
  • Like a lot of things it doesn't care how tall it is just the things inside it.

  • But the chief takeaway is: Inline Elements Do NOT take up all the width so things can apear on the side of an inline element. Be prepared for that.

The fun is when you start taking things that are Blocks and making them act Inline or vice-versa. Go nuts.
