I can't Believe this Didn't Exist Already

In week three, I think, we had the challenge where we wrote the method for Factorial. That was the first time it occurred to me, "Hey I bet, with all these fancy Ruby methods, there is probably already a method that will do factorializing (probably not a word but I'm going with it) for me!"

A little disappointing to see that there wasn't but now it stands as a useful example for building new methods into classes.

Classes, as we all know now, are Object factories. Where do all the little integers come from? Integer heaven? No. the Integer Class. And the Arrays, and the Hashes and the floats and pretty much everything that isn't a method? from; the Array class, the Hash.class and the Float class.

We also know we can create our own classes:

class Shop
end

There you're done, new class. Shop. Easy-peeesy. Adding methods to your new class is also simple:

class Shop

  def my_cool_new_method
    your new method stuff
  end


end

You can also open existing class and add methods to them. Let's put factorial into Integer.

First let's open the Integer class just like before:

class Integer

end

class Integer

def factorial

    total = 1
    num = self
     num.downto(1) do
     total = total * num
    num = num - 1
    end
    total
end

end

Now whenever you are dealing with an integer you can call .factorial as a method and get it.

Okay I should point out that we have not changed ALL OF RUBY. You have to paste this change you want to make about the class Integer, into any Ruby program where you want it to work.
