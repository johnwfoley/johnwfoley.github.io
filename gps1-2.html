The value of Pair Programming

Pair Programing is two coders working together simultaneously. This provides a number of advantanges over programming alone. Including:

  • One Coder can watch for typos, this seems to me the most important part.
  • Two brains working on the problem means less running off to figure out how, "..to do that thing again?"
  • These make for faster devlopment.

Pair Programming at DBC

With each student pairing with each other student we will becoem very comfortable with pairing with lots of people. A valuable skill. Also we will each get an oppurtunity to learn what the other knows. This goes both ways.

Driver vs Navigator

The Driver Types and Composes the code. The Navigator watches and polices typos and logic errors.
